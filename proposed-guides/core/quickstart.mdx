This gets you from zero to a successful sandbox payout. If you can complete it, your
keys, environment, and webhook handling are wired correctly.

## 1. Get your sandbox key

From the Fincra dashboard, copy your **test secret key**. Set it in your shell:

```bash
export FINCRA_SECRET_KEY=<your test secret key>
```

All sandbox calls go to `https://sandboxapi.fincra.com` and move no real money.

## 2. Send a test payout

Send NGN 5,000 to a Nigerian bank account. Replace `business` with your 24-character
business id from the dashboard.

<Tabs>
  <Tab title="cURL">
    ```bash
    curl https://sandboxapi.fincra.com/disbursements/payouts \
      -H "api-key: $FINCRA_SECRET_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "sourceCurrency": "NGN",
        "destinationCurrency": "NGN",
        "amount": 5000,
        "business": "63f9a1b2c4d5e6f7a8b9c0d1",
        "customerReference": "quickstart-001",
        "paymentDestination": "bank_account",
        "beneficiary": {
          "type": "individual",
          "accountHolderName": "Amara Okeke",
          "accountNumber": "0111111111",
          "bankCode": "058",
          "country": "NG"
        }
      }'
    ```
  </Tab>
  <Tab title="Node.js">
    ```javascript

    ```
  </Tab>
  <Tab title="Python">
    ```python

        "https://sandboxapi.fincra.com/disbursements/payouts",
        headers={"api-key": os.environ["FINCRA_SECRET_KEY"]},

            "sourceCurrency": "NGN",
            "destinationCurrency": "NGN",

            "customerReference": "quickstart-001",
            "paymentDestination": "bank_account",
            "beneficiary": {

    ```
  </Tab>
</Tabs>

A `200` returns the payout with `status: processing`. Money hasn't moved yet — payouts
are asynchronous.

## 3. Confirm the result

The final state (`successful` / `failed`) arrives at your webhook as `payout.successful`
/ `payout.failed`. If you haven't set a webhook yet, poll instead:

```bash
curl https://sandboxapi.fincra.com/disbursements/payouts/reference/PO-1a2b3c4d5e \
  -H "api-key: $FINCRA_SECRET_KEY"
```

<Note>
  `customerReference` is your idempotency key. Reusing `quickstart-001` returns
  `DUPLICATE_CUSTOMER_REFERENCE` — use a fresh value for each real payout.
</Note>
